# Contributing Guidelines

Thank you for taking the time to contribute to **Awesome Analysis**! 🎉

Please make sure your contribution follows these guidelines.

## Adding to the list

- **One item per pull request.** This makes review easier and keeps history clean.
- Make sure the resource is **awesome** — high quality, actively maintained, and useful to others.
- Add a single link per line using the format: `` - [Name](link) - Short description. ``
- Keep descriptions **short, clear, and in English** (this is the default language of the project).
- End every description with a period.
- Put new entries in the **correct section** and keep entries alphabetized where the section already is.
- If you add a new section, also add it to the **Contents** table.

## Quality standards

- No broken links.
- No duplicate entries.
- No self-promotion of low-quality or abandoned projects.

## Pull request process

1. Fork the repository and create your branch from `main`.
2. Add your entry following the format above.
3. Make sure the README still renders correctly.
4. Submit the pull request with a clear title and description.
